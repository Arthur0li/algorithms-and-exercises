# 🐍 Python While Exercises

## 🇧🇷 Português (PT-BR)

1. Faça um algoritmo que pergunte ao usuário se ele deseja digitar um número. Enquanto a resposta for diferente de "sim" ou "não", o algoritmo não deve continuar.

2. Faça um algoritmo de validação de idade. Após o usuário digitar a idade, o algoritmo deve verificar se o valor está entre 0 e 100. Enquanto não estiver nesse intervalo, o algoritmo deve continuar solicitando uma nova entrada.

3. Faça um algoritmo que simule um cofrinho. Enquanto o usuário não digitar que deseja sair, o algoritmo deve continuar em loop adicionando os valores informados. Ao final, exiba o total acumulado.

4. Faça um algoritmo que simule o lançamento de um dado e continue executando enquanto o usuário desejar repetir a ação.

5. Faça um algoritmo de adivinhação em que o usuário deve descobrir o número escolhido pelo computador. Enquanto o usuário não acertar, o algoritmo deve continuar em execução.

6. Faça um algoritmo que funcione como um cronômetro. Solicite ao usuário a quantidade de segundos e exiba a contagem até o tempo informado ser atingido.

7. Faça um algoritmo em que o usuário deve descobrir uma senha gerada pelo computador. Enquanto a senha não for descoberta, o algoritmo deve continuar executando e permitindo novas tentativas.

8. Faça um algoritmo que receba valores do usuário e calcule a média. Durante a entrada de dados, o algoritmo deve validar se o valor digitado é numérico. Caso contrário, deve exibir uma mensagem de erro e solicitar novamente.

9. Faça um algoritmo que implemente uma calculadora de juros compostos com suporte a dois idiomas (português e inglês). Enquanto o usuário inserir dados inválidos, o algoritmo deve exibir mensagens de erro e solicitar novas entradas.

## 🇺🇸 English (EN)

1. Write an algorithm that asks the user whether they want to enter a number. While the answer is different from "yes" or "no", the algorithm must not proceed.

2. Write an algorithm for age validation. After the user enters their age, the algorithm must check whether the value is between 0 and 100. While it is not within this range, the algorithm must continue requesting a new input.

3. Write an algorithm that simulates a piggy bank. While the user does not choose to exit, the algorithm should keep running and adding the entered values. At the end, display the total amount saved.

4. Write an algorithm that simulates rolling a die and continues running while the user chooses to keep rolling.

5. Write a guessing game algorithm in which the user must guess the number chosen by the computer. While the user does not guess correctly, the algorithm must continue running.

6. Write an algorithm that works as a timer. Ask the user for the number of seconds and display the count until the specified time is reached.

7. Write an algorithm in which the user must guess a password generated by the computer. While the password is not discovered, the algorithm must continue running and allowing new attempts.

8. Write an algorithm that receives values from the user and calculates the average. During input, the algorithm must validate whether the entered value is numeric. Otherwise, it must display an error message and request a new value.

9. Write an algorithm that implements a compound interest calculator with support for two languages (Portuguese and English). While the user provides invalid inputs, the algorithm must display error messages and request new entries.
