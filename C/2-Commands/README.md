# 🧠 Estruturas de Decisão e Repetição / Decision and Loop Structures

## Português (PT-BR)

1. **Jogo de adivinhação (if-else)**
   Escreva um programa que tente **adivinhar um número entre 1 e 99** pensado pelo usuário.
   A cada tentativa, o usuário deve informar:

   * `=` (igual)
   * `>` (maior)
   * `<` (menor)
     Utilize **`if-else`** para processar as respostas.

2. **Jogo de adivinhação (switch)**
   Reescreva o programa anterior utilizando o comando **`switch`**.
   O programa deve **contar o número de tentativas** e exibir o total ao final.

3. **Média de números positivos**
   Escreva um programa que receba **números inteiros positivos** via teclado.
   Quando um **número negativo** for digitado, o programa deve encerrar e calcular a
   **média dos valores positivos informados**.

4. **Duração de ligação telefônica**
   Escreva um programa que receba dois instantes (início e fim) no formato:

   * horas
   * minutos
   * segundos
     O programa deve calcular o **tempo total da ligação** em horas, minutos e segundos.

5. **Consumo de combustível**
   Receba via teclado:

   * a **distância percorrida em km**

   * a **quantidade de combustível consumido em litros**
     Calcule o consumo em **km/l** e exiba uma mensagem de acordo com a tabela:

   * **Menor que 8 km/l** → Venda o carro!

   * **Entre 8 e 14 km/l** → Econômico!

   * **Maior que 14 km/l** → Super econômico!

## English (EN)

1. **Guessing game (if-else)**
   Write a program that tries to **guess a number between 1 and 99** thought by the user.
   At each step, the user must input:

   * `=` (equal)
   * `>` (greater)
   * `<` (less)
     Use **`if-else`** to handle the responses.

2. **Guessing game (switch)**
   Rewrite the previous program using the **`switch`** statement.
   The program must **count the number of attempts** and display it at the end.

3. **Average of positive numbers**
   Write a program that reads **positive integers** from user input.
   When a **negative number** is entered, the program should stop and calculate the
   **average of the positive values entered**.

4. **Phone call duration**
   Write a program that receives two timestamps (start and end) in the format:

   * hours
   * minutes
   * seconds
     The program must calculate the **total duration of the call** in hours, minutes, and seconds.

5. **Fuel consumption**
   Read from input:

   * **distance traveled (km)**

   * **fuel consumed (liters)**
     Calculate the consumption in **km/l** and display a message according to:

   * **Less than 8 km/l** → Sell the car!

   * **Between 8 and 14 km/l** → Economical!

   * **Greater than 14 km/l** → Very economical!
